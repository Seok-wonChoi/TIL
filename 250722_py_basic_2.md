**리스트**  여러 개의 값을 순서대로 저장하는, 변경 가능한(mutable) 시퀀스 자료형 / **가변 시퀀스**

- 대괄호 [ ] 안에 값들을 쉼표(,)로 구분하여 만듦

**튜플**  여러 개의 값을 순서대로 저장하는 변경 불가능한 시퀀스 자료형

- 소괄호 ( ) 안에 값들을 쉼표(,)로 구분하여 만듦

> 

> 단일 요소 튜플을 만들 때는 반드시 Trailing comma(후행 쉼표)를 사용해야 함

튜플 역시 시퀀스이므로, 인덱싱, 슬라이싱, 길이 확인, 반복 등 공통 기능 모두 사용 가능

- 튜플의 불변 특성을 사용하여 내부 동작과 안전한 데이터 전달에 사용됨
- 다중 할당, 값 교환, 함수 다중 반환 값 등

> 튜플은 데이터의 “안전성과 무결성”을 보장함

**range**  연속된 정수 시퀀스를 생성하는, *변경 불가능한(immutable)* 자료형

- 주로 반복문과 함께 사용되어 특정 횟수만큼 코드를 반복 실행할 때 유용
- 실제로 모든 숫자를 메모리에 저장하는 대신, 시작값, 끝 값, 간격이라는 ‘규칙’만 기억하여 메모리를 매우 효율적으로 사용

range(start, stop, step)

매개변수 : 함수를 정의할 때, 함수가 받을 값을 나타내는 변수

인자 : 함수를 호출할 떄, 실제로 전달되는 값

**딕셔너리(dict)**   key - value 쌍으로 이루어진 순서와 중복이 없는 변경 가능한 자료 / 인덱스 없음, **가변비시퀀스**

- 중괄호 { } 안에 값들이 쉼표(,)로 구분되어 있음
- 값 1개는 키와 값이 상으로 이루어져 있음
- key(키) : 값을 식별하기 위한 고유한 ‘이름표’ (중복 불가)
- Value(값) : 키에 해당하는 실제 데이터
- **각 값에는 순서가 없음**

*Key의 규칙*

- 고유해야 함 (Key는 중복될 수 없음)
- 변경 불가능한(immutable) 자료형만 사용 가능
    - O (가능) : str, int, float, tuple
    - X (불가능) : list, dict

*Value의 규칙*

- 어떤 자료형이든 자유롭게 사용할 수 있음

딕셔너리 값 접근 방법

- Key를 사용하여 해당 Value를 꺼내올 수 있음
- Key에 접근 시 대괄호 [ ] 사용

딕셔너리는 가변비시퀀스

```python
# 딕셔너리 값 추가 및 변경
my_dict = {'apple': 12, 'list': [1, 2, 3]}
# 추가
my_dict['banana'] = 50
print(my_dict)  # {'apple': 12, 'list': [1, 2, 3], 'banana': 50}
# 변경
my_dict['apple'] = 100
print(my_dict)  # {'apple': 100, 'list': [1, 2, 3], 'banana': 50}
```

```python
# 딕셔너리 초기화 2가지 방법
# 1. 하드코딩
my_dict = {"apple":2, "banana":5, "peach":4}
print(my_dict["banana"])
# 2. key로 접근
d = dict() # 빈 딕셔너리
d["apple"] = 2
d["banana"] = 5
d["peach"] = 4

print(my_dict.key()) # key만 출력
print(my_dict.values())
printf(my.dict.ites()) #key와 value 출력

# d = {} << dict
# set() << set
```

**세트(set)**  순서와 중복이 없는 변경 가능한 자료 /  **가변비시퀀스**

세트 표현

- 중괄호 { } 안에 값들을 쉼표(,)로 구분하여 만듦
- 수학에서의 집합과 동일한 연산 처리 가능

*세트의 두 가지 핵심 특징*

1. 중복을 허용하지 않음
    1. 똑같은 값은 단 하나만 존재할 수 있음
2. 순서가 없음
    1. 인덱싱(set[0])이나 슬라이싱(set[0:2])을 사용할 수 없음

합집합 |   차집합 -   교집합 &

```python
# 세트 가변비시퀀스
my_set = {1, 2, 3, 4, 5}
my_set(add(5)
```

**형변환(Type Conversion)**  한 데이터 타입을 다른 데이터 타입으로 변환하는 과

1. 암시적 형변환: 파이썬이 *자동으로* 처리
2. 명시적 형변환: 개발자가 *직접* 지시

**암시적 형변환(Implicit Conversion)**  파이썬이 연산 중에 자동으로 데이터 타입을 변환하는 것

- 정수와 실수의 연산에서 정수가 실수로 변환
- Boolean과 Numeric Type에서만 가능

```python
print(True + False) # True
print(3 + 2.5) # True
```

**명시적 형변환(Explicit Conversion)**  개발자가 변환하고 싶은 타입을 직접 함수로 지정하여 변환하는 것

```python
num = '3.5' # 바로 int() >> error
num = int(float(num)) # int는 버림
print(num) # 3
```

### 비교 연산자

두 값을 비교하여 그 관계가 맞는지 틀리는지 True 또는 False로 반환

**== 연산자**

- 값(데이터)이 같은지 비교
- 동등성(equality)
- 예를 들어, 1 == True의 경우 내부적으로 True를 1로 간주할 수 있으므로 True 결과가 나옴

**is 연산자**

- 객체 자체가 같은지 비교
- 식별성(identity)
- 두 변수가 완전히 동일한 객체를 가리키는지, 즉 메모리 주소가 같은지 확인할 때 사용

```python
print(2 == 2.0) # True           
																     print(1 is True) # False
print(2 != 2) # False            
																     print(2 is 2.0)  # False
print(1 == True) #True
```

is 대신 ==를 사용해야 하는 이유

- 결론 : is는 ‘정체성’을, ==는 ‘가치’를 비교하기 때문

is 연산자는 언제 사용하는가? 

- 주로 싱글턴 객체를 비교할 때

싱글턴(singleton) 객체란?

- 특정 값에 대해 파이썬 전체에서 *단 하나의 객체만 생성되어 재사용*되는 특별한 객체
- 파이썬의 대표적인 싱글턴 객체: None, True, False

```python
a = [1, 2, 3]
b = [1, 2, 3]
b = a
print(a is b) # True
```

### 논리 연산자

여러 개의 조건을 조합하거나, True/False 값을 반대로 뒤집을 때 사용 (and, or, not이 대표적)

**단축 평가**  논리 연산에서 두 번째 피연산자를 평가하지 않고 결과를 결정하는 동작

거짓으로 취급되는 값

- False, 숫자 0, 빈 문자열 “ “, 빈 리스트[ ], None 등

참으로 취급되는 값

- True 그리고 ‘거짓’이 아닌 모든 값

단축 평가를 하는 이유

- 코드 실행을 최적화하고, 불필요한 연산을 피할 수 있도록 함

```python
a and b # 둘 다 참이어야 함
a or b # 둘 중 하나라도 참이면 참

print(0 or 1 or 3) # 1
print(2 and 3 and 7) # 7
print(7 and 0 and 3) # 0
```

### Trailing Comma (후행 쉼표)

컬렉션의 마지막 요소 뒤에 붙는 쉼표

- 각 요소를 별도의 줄에 작성
- 마지막 요소 뒤에 trailing comma 추가
- 닫는 괄호는 새로운 줄에 배치

![image.png](attachment:90cbc752-19ce-4b27-b54a-aafe2c1beba3:image.png)

Trailing Comma 장점

1. 가독성 향상
    1. 각 줄이 동일한 패턴을 가짐
    2. 코드 리뷰가 용이함
2. 유지보수 용이성
    1. 항목 추가/제거가 간단
    2. 실수로 인한 구문 오류 방지
